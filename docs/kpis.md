# KPIs — Hospital / Clinic Management System

## Key Performance Indicators

| ID | KPI | Description | Target | Current | Status |
|----|-----|-------------|--------|---------|--------|
| KPI-01 | Defect Rate | Number of bugs found per sprint | < 3 bugs/sprint | 1 bug/sprint | ✅ On Target |
| KPI-02 | Lead Time | Time from task creation to deployment | < 5 days | 3 days | ✅ On Target |
| KPI-03 | Deployment Frequency | How often code is deployed to production | 2x per week | 3x per week | ✅ Exceeding |
| KPI-04 | Response Time | Average API endpoint response time | < 500ms | 620ms | ⚠️ Needs Work |
| KPI-05 | System Availability | Percentage of time system is online | > 99% | 99.5% | ✅ On Target |

## KPI Definitions

### KPI-01 Defect Rate
Measures the quality of code delivered each sprint. A low defect rate means the team is writing clean, well-tested code.

### KPI-02 Lead Time
Measures how fast the team delivers value. Shorter lead time means faster delivery to users.

### KPI-03 Deployment Frequency
Measures how often the team deploys. Higher frequency means smaller, safer releases.

### KPI-04 Response Time
Measures how fast the system responds to user requests. Target is under 500ms for good user experience.

### KPI-05 System Availability
Measures system uptime. Target is 99% or higher to ensure patients can always access the system.
