# Support Plan — Hospital / Clinic Management System

## Support Process

### Issue Reporting Process
1. User reports issue via email or support form
2. Support team logs issue in defect-log.md
3. Issue is assigned a Bug ID (e.g., BUG-002)
4. Severity is assessed (Low / Medium / High / Critical)
5. Issue is assigned to the appropriate developer
6. Fix is developed on a `bugfix/<name>` branch
7. Fix is tested by QA (Mitch Dumdum)
8. Fix is merged via Pull Request
9. User is notified of resolution

## Response Times

| Severity | Description | Response Time | Resolution Time |
|----------|-------------|---------------|-----------------|
| Critical | System down, data loss | 1 hour | 4 hours |
| High | Major feature broken | 4 hours | 24 hours |
| Medium | Minor feature issue | 24 hours | 3 days |
| Low | Cosmetic or minor bug | 3 days | 1 week |

## Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| Cannot log in | Reset password, check credentials |
| Appointment not saving | Clear browser cache, retry |
| Page not loading | Check internet connection, refresh |
| Wrong billing amount | Contact receptionist to manually adjust |

## Escalation Process
1. **Level 1** — Support team handles common issues
2. **Level 2** — Developer assigned for technical bugs
3. **Level 3** — Product Owner (Ryan Hermoso) for critical decisions
